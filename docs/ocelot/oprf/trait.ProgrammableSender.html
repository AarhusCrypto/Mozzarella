Run the OPPRF, with (x, y) values given by points being programmed into the PRF. The value npoints is an upper-bound on the length of points. The value t must be set to 1, otherwise we return Error::InvalidInputLength.

Note that npoints often needs to be much larger than the length of points, as otherwise the sender won't be able to uniquely map the x values. If this is the case, we return Error::Other.
